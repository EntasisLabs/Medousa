# Medousa 0.5.0 — Vault, Versions, Liquid feeds

Living implementation notes for the 0.5.0 train. Product language for Git-backed history is **Versions** (Advanced Git stays secondary).

## Pillars

1. **Editor polish** — Live/Build coherence, conflict/proposal bars, trash restore.
2. **Versions (Git-backed)** — off by default (`vault_git_enabled`); daemon-owned subprocess Git.
3. **Scripts → code editor shell** — Grapheme first; thin `CodeEditorShell` for other languages.
4. **Liquid viz** — snapshot timeline+carousel; ```feed``` fence for Stasis last-good results.

## Daemon APIs

| Route | Purpose |
|-------|---------|
| `GET /v1/vault/git/detect` | binary path + version + platform hint |
| `GET /v1/vault/git/status` | enabled, is_repo, branch, dirty |
| `POST /v1/vault/git/enable` | flip flag; optional init |
| `POST /v1/vault/git/init` | `git init` + `.gitignore` |
| `POST /v1/vault/git/install` | Windows MinGit; macOS/Linux guidance |
| `GET /v1/vault/git/log` | history |
| `POST /v1/vault/git/commit` | Save version |
| `POST /v1/vault/git/restore` | restore path from commit |
| `GET /v1/vault/git/diff` | note patch vs commit |
| `GET|POST|DELETE /v1/vault/git/worktrees` | Advanced Git worktrees |
| `GET /v1/vault/trash` | list `.trash/` |
| `POST /v1/vault/trash/restore` | restore a trashed note |
| `GET /v1/feeds/{id}/latest-good` | last successful feed payload for liquid |

## Settings

Settings → **Versions**: off/on (mirrors Shell tools opt-in). Install Git / Start versioning when needed.

## Liquid

- Timeline `layout: snapshot` — horizontal rail + synced carousel (Live surface).
- ```feed``` fence — hydrate last-good with datatype `md|text|json|csv|image`.

## Out of scope

- Non-Git version backends, bundled macOS Git.app, multi-user collab, Scripts IDE (debugger/extensions).
