# Medousa 0.3.2

**Vault edits that stay yours. Compares that keep rendering. Loose markdown that opens.**

A reliability release for the LME / vault editor — closes the race that could wipe an active note when creating another, and fixes Live freezes, external markdown tabs, and broken compare embeds.

## Highlights

### Note create no longer eats the open note
Cold opens take the write lease before applying body. Stale remounts and autosaves cannot PUT a new template onto the note you were editing.

### Live stays calm under paste and tab switches
Destroy-time TipTap flushes are gone. Empty / frontmatter-only notes no longer spin the Live hydrate loop into a freeze.

### External markdown + compare embeds
Opening a loose `.md` binds an LME tab immediately and renders once the file read finishes. Duplicate compare axis/entity labels get unique ids so one bad fence cannot blank everything after it.

### Context menu stays responsive on Windows
Cut / copy / paste from the editor menu use timed clipboard reads and a body-portaled menu so WebView2 permission hangs cannot freeze the whole shell.

---

## Upgrade notes

- No migration. Existing vault notes and loose files keep working.
- If you hit a wipe on an older build, check disk / backups — there is still no undo for a successful wrong-path PUT from before this fix.
