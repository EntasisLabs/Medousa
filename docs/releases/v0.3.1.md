**Quieter LME. Schedules that read as owned things. Vault edits that stick.**

## Highlights

### LME rails that feel like a workbench
Agents, flows, schedules, and history share the same quiet language — less uppercase chrome, hover-reveal actions, liquid docks. History search expands inline in the footer like Files / Scripts / Decks.

### Schedules as first-class tabs
Open a schedule in LME. Name and one health story lead; cron and ids live under Details. **New schedule** is a progressive popover (title · what · when · deliver to), not a rail takeover.

### Vault & shell reliability
Right-click context menu in the note editor. Note buffers and save queue hold state across tab switches. Shell tab hit-targets no longer hide under the pointer or fight titlebar actions.

### Fixes you’d notice
Live kanban drag restored. Editor truncation and script-action overlay clicks fixed. Split-rail / schedule pane boundaries behave.

---

## Upgrade notes

- No migration. Existing schedules, flows, and notes keep working.
- If a schedule tab title still looks like a truncated prompt, reopen it after upgrade — labels now prefer display names / short human titles.
